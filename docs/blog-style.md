# ABBEY'S TOYS 部落格風格指南（Blog Style Guide）

阿北玩具堂 abbeystoys.com 部落格的寫作規範。所有新文章（含 Claude 代筆的草稿）
都必須符合本文件。**首要範本：`/blog/night-lords-first-claw`（阿北 2026-07-13
親筆重寫，這篇就是標準答案）**；短篇購物指南另見 `/blog/joytoy-warhammer-40k-guide`。

## 定位與聲音

- **作者人設：阿北** — 「身為一名熱愛《Warhammer 40k》的老粉絲」——
  不是店員在推銷，是老玩家在跟同好聊天。開場直接對讀者說話
  （「各位戰錘同好」），敢下判斷、敢開玩笑（「那你可找錯黑圖書館的
  書架了」），結尾像朋友道別（「保持黑暗，朋友們。」）。
  繁體中文（zh-TW）、台灣用語。
- **讀者**：戰鎚同好與收藏者。把讀者當自己人，不當小白——可以深、
  可以長，重點是讓人讀完想去看書、想留言、想收模型。
- **目的**：內容自然帶動收藏與討論。**內文不放銷售 CTA**——文章的
  tags 會自動在文末長出「🛒 想收這個派系？」商品連結框，讓內容歸內容、
  導購歸導購。

## 結構（深度解析文體 — 預設格式）

1. **標題**：「派系：主題——副題」（例：`午夜領主：永存的恐怖——第一爪深度解析`）。
   模板會自動加 `| Blog - ABBEY'S TOYS 阿北玩具堂` 後綴。
   測試會檢查 title 長度與單一 h1 —— 內文**不可**使用 `#` 一級標題。
2. **開場**：1～2 段世界觀鉤子＋老粉自我介紹，開頭第一段純文字
   （前 155 字會變成 meta description）。
3. **導讀章節**（`## 《書名》導讀：…`）：這本書在講什麼、為什麼好看，
   評論作者筆觸——像書評，不像百科。
4. **深度解析章節**（`## 深度解析：…`）：角色逐一介紹。
   格式：**粗體名稱行**（`**Talos Valcoran（塔洛斯·瓦爾科蘭）——稱號**`）
   ＋一段側寫。專有名詞**英文在前、中文在括號**
   （`First Claw（第一爪）`、`Soul Hunter（魂獵者）`）。
5. **收束章節**（`## 為什麼…令人難忘`）：講這個故事為什麼值得、
   角色為什麼留在心裡——觀點，不是摘要。
6. **結尾**：向讀者提問＋邀請留言（「你最喜歡…？歡迎在留言區分享」）
   → 派系口號附中文（`Ave Dominus Nox（黑暗主宰萬歲）`）
   → 括號簽名句（「（戰錘老粉分享。保持黑暗，朋友們。）」）
   → `---` 分隔線後列**參考資料**（來源名稱即可，不用連結）。
7. **長度**：深度解析 1500～2500 中文字；購物指南／入門短文 500～900 字。

## 連結（內部 SEO）

- 至少一個 `/codex/<slug>` 圖鑑連結，融在正文裡、別打斷語氣
  （例：`[Night Lords（午夜領主）](/codex/night-lords)`）。
- **不放商品 CTA**：把派系英文 tag（如 `Night Lords`）放進文章 tags，
  文末的「🛒 想收這個派系？」框會自動連到商品；只有購物指南類短文
  才在內文放 `/products` 連結。
- 相關舊文互連（`/blog/<slug>`）——**只連已發佈的文章**，連到草稿會 404。
- 適時連 `/quiz`（原體測驗）。

## 劇透政策（2026-07-13 更新）

- 深度解析可以進入劇情，但**必須先給明確的粗體劇透警告**，
  說清楚雷到哪本書，讓讀者自己選擇：

  > **為了尊重讀者，以下會有重大劇透，特別是第一本《Soul Hunter（魂獵者）》。
  > 已讀過的可以安心繼續，沒讀過的請自行斟酌。**

- 即使警告過，也**不逐一點名生死與結局**——講到「代價」「耗損」即可，
  把最後一頁留給書。
- 無劇透的短文（入門／指南）維持「鉤子不給答案」的寫法。
- 想額外把重雷收進摺疊框仍可用劇透語法：
  `:::spoiler 標題（重大劇透）…:::`（摺疊框）、`||行內短雷||`（點擊顯示）。
  meta description／og／schema 會自動用去雷版本。

## 命名與用詞

- 專有名詞第一次出現：**English（中文）**——英文在前、中文在括號
  （`First Claw（第一爪）`）；之後可只用中文。書名保留英文
  （《Soul Hunter（魂獵者）》）。
- 中文譯名**以店內商品名為準**（例：康拉德·科茲、賽維塔、暗夜猛禽），
  商品沒有的再用台灣社群通用譯法；查 `/codex` 與 POS `tag_glossary`。
- 品牌寫 **JOYTOY**（全大寫）。比例寫 1/18、1/25。
- 產品 tag 保持英文 key（`Night Lords`），顯示時模板會套中文標籤。

## 資料格式（storefront_posts）

- `type='blog'`、`slug` 全小寫連字號英文、`title` 中文、`body` markdown。
- `extra` JSON：`{"author": "阿北", "excerpt": "50~70字帶懸念的摘要", "tags": [中英混合 3~5 個]}`。
- `published_at` = `YYYY-MM-DD`；**草稿用 `is_published=0`**，
  阿北在 POS 審完改 1 才上線。
- 發佈流程：冪等 upsert script（`ON CONFLICT(type,slug) DO UPDATE`）→
  本地 DB 測試 → 本地 Flask 檢查渲染 → 跑在 VM 的 prod DB。

## 原創性（紅線）

- 事實與設定可以參考 wiki、小說、影片與逐字稿；**文字必須全部原創**，
  不得改寫抄襲任何來源的句子（包含達奇影片的旁白）。
- 引述名句限一兩句、加引號、屬於原作台詞（如「Ave Dominus Nox」）。
- 文末**參考資料**列出用過的來源名稱（Lexicanum、Goonhammer…），
  不需要網址，也不要留 `[web:N]` 這類工具產生的殘渣。

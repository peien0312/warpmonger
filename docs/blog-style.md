# ABBEY'S TOYS 部落格風格指南（Blog Style Guide）

阿北玩具堂 abbeystoys.com 部落格的寫作規範。所有新文章（含 Claude 代筆的草稿）
都必須符合本文件。範本：`/blog/horus-heresy-intro`、`/blog/night-lords-intro`。

## 定位與聲音

- **作者人設：阿北** — 台灣玩具店老闆，也是資深戰鎚玩家。內行但不掉書袋，
  像朋友在店裡跟你聊收藏。繁體中文（zh-TW）、台灣用語。
- **讀者**：JOYTOY 收藏者與戰鎚新人。假設讀者「常聽到這些名詞但不熟」，
  每篇都要能獨立看懂，不需要先讀過其他文章。
- **目的**：每篇文章最終都要自然地導向收藏 —— 介紹背景是為了讓讀者
  「想收這個派系／角色」，不是寫維基。

## 結構

1. **標題**：15～25 個中文字，格式偏好「主題：副題」
   （例：`午夜領主入門：以恐懼為武器的第八軍團`）。模板會自動加上
   `| Blog - ABBEY'S TOYS 阿北玩具堂` 後綴，標題本身不要再塞店名。
   測試會檢查 title 長度與單一 h1 —— 內文**不可**使用 `#` 一級標題。
2. **開頭段**：純文字（不放連結、粗體、圖），因為前 155 字會直接變成
   meta description。一段講完「這篇要帶你認識什麼、為什麼值得看」。
3. **章節**：2～4 個 `##` 段落，每段 2～4 句。小標題用有畫面感的短語
   （「恐懼就是武器」優於「軍團戰術介紹」）。
4. **收藏段**：倒數第二段固定是「收藏○○」／「收藏對應」——連結到
   JOYTOY 對應商品與系列頁。
5. **結尾**：`👉 逛逛[○○系列](/products?tag=...)` CTA，加一句相關的
   延伸閱讀連結，最後單獨一行放派系口號
   （帝國：`For the Emperor！`；午夜領主：`Ave Dominus Nox！`；
   依派系選用，格調一致）。

## 連結（內部 SEO，每篇必備）

- 至少一個 `/codex/<slug>` 圖鑑連結（錨文字用「中文名 英文名」格式，
  例：`[里曼·魯斯 Leman Russ](/codex/leman-russ)`）。
- 至少一個商品連結：系列頁 `/products?tag=<英文Tag>`（tag 用英文 key，
  URL encode 空格）或 `/products?category=<slug>`，加上 1～2 個明星單品
  的商品頁連結。
- 相關舊文互連（`/blog/<slug>`），新系列文章彼此互連。
- 適時連 `/quiz`（原體測驗）與 `/faq`。

## 劇透政策（嚴格遵守）

- **正文絕不揭露**角色生死、身分反轉、結局。寫「鉤子」不寫「答案」：
  用「他藏著一個連隊長都不知道的祕密——讀到揭曉那一刻你會想重讀他每句台詞」
  這種寫法吊胃口，把答案留給書。
- 世界觀底層設定（如荷魯斯之亂的結局）可以講，**小說的劇情轉折**不行。
- 真的值得討論的重雷內容，用劇透語法收進摺疊框，且標題註明「重大劇透」：

  ```markdown
  :::spoiler 三部曲結局走向（重大劇透）
  被隱藏的段落（就算展開了也點到為止，不逐一點名生死）。
  :::

  行內短雷用 ||雙豎線包起來|| （點擊顯示）。
  ```

## 命名與用詞

- 專有名詞第一次出現：**中文（English）**，之後只用中文。
- 中文譯名**以店內商品名為準**（例：康拉德·科茲、賽維塔、暗夜猛禽），
  商品沒有的再用台灣社群通用譯法；查 `/codex` 與 POS `tag_glossary`。
- 品牌寫 **JOYTOY**（全大寫）。比例寫 1/18、1/25。
- 產品 tag 保持英文 key（`Night Lords`），顯示時模板會套中文標籤。

## 資料格式（storefront_posts）

- `type='blog'`、`slug` 全小寫連字號英文、`title` 中文、`body` markdown。
- `extra` JSON：`{"author": "阿北", "excerpt": "50~70字帶懸念的摘要", "tags": [中英混合 3~5 個]}`。
- `published_at` = `YYYY-MM-DD`；**草稿用 `is_published=0`**，
  阿北在 POS 審完改 1 才上線。
- 發佈流程：冪等 upsert script（`ON CONFLICT(type,slug) DO UPDATE`）→
  本地 DB 測試 → 本地 Flask 檢查渲染 → 跑在 VM 的 prod DB。

## 原創性（紅線）

- 事實與設定可以參考 wiki、小說、影片與逐字稿；**文字必須全部原創**，
  不得改寫抄襲任何來源的句子（包含達奇影片的旁白）。
- 引述名句限一兩句、加引號、屬於原作台詞（如「Ave Dominus Nox」）。

## 長度

- 正文 500～900 中文字。寧可短而勾人，不要長而完整——
  「想知道更多」的讀者會去點 codex、去買書、去下單。

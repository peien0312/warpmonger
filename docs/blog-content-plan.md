# 部落格內容規劃（faction series）

搭配 `blog-style.md`（怎麼寫）使用；這份講**寫什麼、排程與審稿流程**。

## 每個派系是一個小系列

以午夜領主為原型（intro → 角色 → 群像），每個派系規劃 2～4 篇：

1. **軍團／派系入門**（`<faction>-intro`）— 是誰、為何迷人、怎麼收藏。
2. **原體篇** — 把原體當「人物傳記的鉤子」寫：他最矛盾的一個決定、
   最有名的一句話、與哪個兄弟的恩怨。
3. **人物篇** — 名角色（軍團第一連長、名小說主角…），連到對應模型。
4. **事件篇** — 大事件（普羅斯佩羅焚城、鐵籠、卡爾斯…）當「故事預告片」寫。

### 跟 codex 的分工（重要）

codex 是**百科**：中立、完整、給查資料的人。
blog 是**說書**：有阿北的聲音、有懸念、有「為什麼你會愛上這個派系」，
結尾永遠通向商品或書。判斷法：一段文字如果搬進 codex 毫無違和，
就代表寫得太像 codex——重寫成有觀點的故事（用第二人稱、用反問、
用「店裡最常被問…」的場景切入）。blog 文中大量連 codex 沒問題，
那正是分工：故事在 blog，查證在 codex。

## 審稿工作流程（2026-07-13 v2 — 四狀態）

狀態欄位 `review_status`（POS 內容列表有對應徽章），`is_published=1` 即為
「已發佈」並蓋過其他標籤：

| 狀態 | 徽章 | 誰觸發 | 意義 |
|---|---|---|---|
| `draft` | 草稿 | Claude 產稿 | AI 生成，待阿北初審 |
| `reviewed` | 待重寫（有意見） | 阿北在審稿備註寫意見並存檔（自動切換） | 有新意見，等 Claude 重寫 |
| `ready` | 待放行（已重寫） | Claude 重寫完 upsert | 已依意見重寫，等阿北複審（可再留言打回 reviewed） |
| （released） | 已發佈 | 阿北勾「發佈到網站」 | 上線 |

規則：
- **審稿備註是追加式的紀錄，永不清空**——新意見寫在最上面；Claude 重寫後
  只「追加」一行處理標記（`── Claude YYYY-MM-DD：…──`），舊意見全保留。
- Claude 撈工作清單：`review_status='reviewed'`（不限 type，codex 也適用）。
- 重寫不動 `is_published`；發佈永遠是阿北手動勾。
